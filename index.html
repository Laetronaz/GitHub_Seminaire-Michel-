Introduction

Notions couvertes

Dans ce séminaire, vous appliquerez les notions de contrôle de sources avec l’aide d’un serveur git distant. L’Apprentissage passera par l’inscription au site GitHub, a la création de répertoire git, la synchronisation des fichiers à travers git, l’envoie d’une nouvelle version d’un répertoire git, la visualiser des statistique des répertoires vous appartenant, la création et l’utilisation de branches pour un même répertoire et finalement la création et la gestion d’une organisation GitHub.

Connaissances technologiques pré-requises

Pour suivre une partie de ce séminaire une connaissance de linux et des lignes de commandes, plus précisément des distributions basé Debian, est nécessaire pour une partie du séminaire. Il est préférable de connaitre à niveau moyen les concepts du contrôle de sources et/ou avoir utilisé certains de ses logiciels par le passé.Il est aussi nécessaire d’avoir des bases sur le concept d’arborescence des fichiers.

Outils technologiques utilisés

  • git
  • GitHub
  • Un navigateur web de votre choix
  • Le terminal système

Laboratoires

Laboratoire 01 : Inscription à Git Hub

  • Rendez- vous sur le site https://github.com/
  • Inscription:
  • Sur ce site vous devriez voir un formulaire d’inscription
  • Dans le champ username entrez le nom d’utilisateur sous lequel vous désirez être identifie
  • Dans le champ email entrez votre adresse email (valide)
  • Puis dans le champ password entrez un mot de passe (au minimum 7 caractères avec une lettre et un chiffre)
  • Une fois les 3 champs cochés par un crochet vert appuyé sur le bouton Signup to GitHub
  • Choix du plan :
  • Choisissez le plan que vous désirez par défaut le plan a 0$ (dernier de la liste)
  • Cliquez sur le bouton vert Finish Sign up en bas de la page

Félicitation vous avez maintenant un compte Github!

Laboratoire 02 : Installation du/des logiciels

Pour Windows Installer l’application GitHub
  • Accéder à la page https://windows.github.com/
  • Cliquer sur Download GitHub
  • Éxécuter GitHubSetup.exe
  • Si un pop-up apparait cliquez sur Run et Install
  • Une fois l’installation terminée l’application ouvrira d’elle-même
  • Dans log in entrez les informations de votre compte et appliquer
  • Cliquez sur continue pour enregistrer les informations de configurations
  • Accéder à la page http://git-scm.com/
  • Cliquez sur download
  • Exécuter le setup
  • Cliquez sur next jusqu’à la fin.
Pour Ubuntu installer Git
  • Allez dans votre terminal Ubuntu (Chercher Terminal ou faites ctrl+alt+t)
  • Entrez la ligne suivante : sudo apt-get install git
  • Lorsqu’il vous le sera demandé entrez votre mot de passe et y lors de la question oui ou non
  • Il faut maintenant connecter le répertoire GitHub à notre ordinateur git clone https://github.com/username/repertoire.git
  • Il est possible de sauvegarder les informations du user et du email pour les prochaines opérations
  • Le username : $ git config --global user.name "Votre Username"
  • Le email: $ git config --global user.email "votre@email.com"

Vous avez maintenant accès à votre répertoire et vos informations seront sauvegarder quand vous ferez des opérations sur le répertoire github.

Laboratoire 03 : Mise en place du premier répertoire

Mettre en place son premier répertoire (sur github)

  • Cliquez sur votre nom en haut de la page.
  • Une fois votre profil, choisissez l’onglet repositories
  • Puis cliquez sur le bouton Vert New
  • Dans le champ repository name entrez le nom que vous désirez donner à votre répertoire
  • Dans le champ Description vous pouvez entrer une description de ce que va contenir ce répertoire. (optionnel)
  • Vous devez ensuite choisir une visibilité entre public et privé nous prendront dans ce laboratoire la visibilité publique.
  • Ensuite sélectionnez Add a License et choisissez l’option MIT License
  • Cocher l’option Initialize this repository with a README
  • Cliquez enfin sur le bouton vert Create Repository

Sécuriser le répertoire

  • Cliquer sur l’option Settings à droite du répertoire.
  • Cocher l’option Restrict editing to collaborators only
  • Cela empêchera les autres utilisateurs de GitHub d’éditer le répertoire sans que vous ne les nommiez Collaborateur.

Récupérer le répertoire en local

  • Cliquez sur le + en haut à gauche
  • Choisissez clone sélectionnez votre compte et le répertoire que vous voulez importer

Laboratoire 04 : Prise en main du répertoire

Pour Windows avec application Git Hub :

Créer un répertoire à partir de fichiers courants

  • Cliquer sur +
  • Choisissez Create entrez un nom et choisissez le répertoire dans votre arborescence
  • Pour l’envoyer sur GitHub cliquer sur Publish Repository en haut à droite

Pour créer ou modifier un fichier et le PUSH a votre projet GitHub

  • Allez dans le repertoire cloner dans votre arborescence
  • Créez un fichier example : new_or_existing_file.extention
  • Cliquer droit sur le répertoire et sélectioner l’option git Gui
  • Vous verrez une liste des modifications de votre répertoire
  • Cliquer sur State Change pour accepter les changements
  • Écriver un message de Commit example username : La modification
  • A partir d’ici 2 solution sont possible simplement cliquer sur Push et entrez les informations du compte GitHub ou aller sur l’interface GitHub client et cliquer sur Sync en haut à droite.

Pour synchroniser les informations de la BRANCH et PULLER tous les nouveau changements

  • Aller sur github.com et cliquer sur le nouveau fichier ajouter puis sur le symbole de crayon afin d’éditer le fichier à partir de l’interface web.
  • Rajouter du texte et en bas de page ajouter un message de commit et optionnellement une description
  • Choisissez commit directly sur master branch et cliquer sur le bouton vert Commits changes
  • Retourner sur le client de GitHub, vous verrez un symbole apparaitre sur le bouton sync, cliquer dessus.

Voir tous les changements de la version locale

  • cliquer droit et choisissez git gui tous les changements non enregistré apparaitront dans la partie du haut Voir toutes les activités sur repertoire GitHub

  • Accéder à votre Compte GitHub et cliquer sur le nom de votre répertoire

  • Cliquer sur le mot commits en haut de la page il vous permettra de sélectionner les commits et de voir toutes les modifications apportés lors de ceux-ci.
Pour Ubuntu avec Git :
  • Note : Pour transformer un répertoire existant en répertoire git il suffit de faire git init une fois à l’intérieur du répertoire Pour créer ou modifier un fichier et le PUSH a votre projet GitHub
  • Entrez dans le répertoire git cd nomdurépertoire (TRÈS IMPORTANT!!!)
  • Créer un fichier : touch new_or_existing_file.extention 
git add new_or_existing_file.extention 
git commit
git push origin master

Pour synchroniser les informations de la BRANCH et PULLER tous les nouveau changements

  • Aller sur github.com et cliquer sur le nouveau fichier ajouter puis sur le symbole de crayon afin d’éditer le fichier à partir de l’interface web.
  • Rajouter du texte et en bas de page ajouter un message de commit et optionnellement une description
  • Choisisez commit directly sur master branch et cliquer sur le bouton vert Commits changes
git fetch
git merge origin/master
  • Visualiser toutes les requetes FETCH et PULL à distance que vous avez demandé git remote –v
  • Voir les changements récents git status
  • Visualiser tous les changements sur votre FORK git log
  • Une liste détaillée de tous les changements qui ont été commits sur votre FORK git log –stat
  • Une liste des changements avec couleurs git log --graph --full-history --all --color --pretty=format:"%x1b[31m%h%x09%x1b[32m%d%x1b[0m%x20%s"

Laboratoire 05 : Utiliser les branches

Pour Windows :

Visualiser les branches disponibles pour le répertoire git

  • Sur la page GitHub le bouton branch :
  • Sur l’application GitHub quand un répertoire est sélectionné le bouton texte en haut de l’historique.
  • Avec Git clic droit sur le répertoire et option branch
  • Sur Git Gui dans branch et Checkout

Créer une nouvelle branche et l’envoyer à GitHub

  • Sur GitHub directement et synchroniser les informations (bouton sync)
  • Sur l’application Git sur le bouton text branch et faire publish
  • Avec Git Gui Branch Create et remplir les champs nécessaire, puis faire un push (façon plus optimale) Visualiser de nouveau vos branches

Changer de branche (Checkout)

  • Par l’interface GitHub en haut de la page
  • En cliquant droit avec Git et choisissant la branche sur le répertoire

Modifier localement votre branche et sauvegarder les changements

  • Supprimer un fichier d’une des branches
  • Faire un push comme appris dans le Lab03

Si un update est nécessaire sur une branche

  • On peut synchroniser avec l’Application client
  • Git Gui cliquer sur remote puis fetch from et finalement origin.

Supprimer une branche en local

  • A partir de Git Gui dans branch delete selectionner la branche et cliquez sur delete
  • Sur l’application client de GitHub cliquer sur les branches et cliquez sur manage, puis cliquer sur la poubelle sur la branche que vous désirez supprimer.

Supprimer une branche sur gitHub

  • Si vous avez utilisé Git Gui il suffit de faire un nouveau push.
  • Si vous avez utilisé GitHub il suffit de faire Sync
  • Si vous n’avez encore rien faire vous pouvez utiliser l’interface web pour supprimer la branche sur GitHub directement. Dans le répertoire branches puis all branches.
Pour Ubuntu

Visualiser les branches disponibles pour le projet

git branch
  • Vous ne devriez voir qu’une seule branche

Créer une nouvelle branche et l’envoyer à GitHub

git checkout –b nomdevotrenouvellebranche
git push origin nomdevotrenouvellebranche

Visualiser de nouveau les branches disponibles

  • git branch
  • une seconde branche devrait être disponible

PUSH les changements de vos commits dans votre branche

  • git push origin nomdevotrebranche
  • Mettre à jours votre branche à partir du dépôt Github
  • Modifier à partir de l’interface web la nouvelle branche sélectionner votre nouvelle branche et supprimer le fichier créer précédemment. (cliquer dessus puis cliquer sur le symbole de poubelle)
git fetch 
git merge origin/nomvotrebranche

Changer de branche

git checkout master

Supprimer une branche

$ git branch –D nomdevotrebranche
$ git push origin :nomdevotrebranche

Laboratoire 06 : Les organisations (Optionnel)

Durant ce laboratoire vous apprendrez comment créer et gérer les diverses aspects d’une organisation sur GitHub. Une fois ce laboratoire terminé, vous serez en mesure de créer des répertoires, gérer les membres d’une organisation, gérer des groupes et les permissions de ceux-ci.

Créer une organisation

  • Allez sur votre GitHub cliquez sur le + et cliquer new organisation
  • Entrez un nom et un email
  • Sélectionner l’option Open Source
  • Cliquez create puis organization

Inviter des membres

  • Cliquez haut à droite de la page de l’organisation sur le bouton invite someone
  • Entrez le nom d’utilisateur de la personne
  • Sélectionnez le groupe auquel il appartiendra

Créer et gérer une équipe

  • Cliquer à droite dans le menu Team sur Create a new Team
  • Entrez le nom et la description de l’équipe puis choisissez les accès Read, Write ou Admin
  • Cliquez sur Create Team

Créer un répertoire

  • Cliquez sur New Repository sur la page de l’organisation
  • Entrez un nom et une description
  • Cocher une visibilité
  • Ajouter une licence MIT (juste au cas)
  • Cliquez sur create repository

Affecter un répertoire a un groupe

  • Cliquez sur le groupe en question
  • Cliquez sur repositories
  • Dans la barre de recherche add Repositories entrez le nom du répertoire
  • Entrez votre mot de passe dans le pop-up

Laboratoire 07 envoie d’un répertoire local vers Git Hub

Pour windows :

  • Cliquez droit sur le répertoire que vous désirez envoyer et cliquez git init here
  • Allez sur l’application gitHub cliquez sur le +
  • Choissisez add et entrez le chemin vers le repertoire
  • Cliquez ensuite add repository
  • Sur le nouveau repository remplissez les champs des uncommits changes
  • Cliquez sur commit changes
  • Envoyer le répertoire a github en cliquant sur publish repository en haut à droite de l’écran.
  • Choisissez l’endroit où vous voulez faire ce publish : votre compte ou une organisation à laquelle vous appartenez.
  • Cliquez sur publish
  • Finalement le repertoire est publier mais pas le contenu, il suffit de faire publish en haut à droite pour que le contenu soit envoyer à GitHub
Pour Ubuntu
  • Utilizez cd pour vous déplacer dans le répertoire
  • Entrez la commande git init
  • Faites un git add pour les fichiers interne s’il y a lieu
  • Faites un git commit
  • Créez un répertoire vide sur github et copier l’url de clui-ci
  • Créer un accès à distance git remote add origin url(celui récupérer plutôt)
  • Vérifier le remote git remote –v
  • Finalement envoyer les changements local vers la destination : git push origin master
