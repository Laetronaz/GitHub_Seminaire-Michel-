Introduction

Notions couvertes

Dans ce séminaire, vous appliquerez les notions de contrôle de sources avec l’aide d’un serveur git distant. L’Apprentissage passera par l’inscription au site GitHub, a la création de répertoire git, la synchronisation des fichiers à travers git, l’envoie d’une nouvelle version d’un répertoire git, la visualiser des statistique des répertoires vous appartenant, la création et l’utilisation de branches pour un même répertoire et finalement la création et la gestion d’une organisation GitHub.

Connaissances technologiques pré-requises

Pour suivre une partie de ce séminaire une connaissance de linux et des lignes de commandes, plus précisément des distributions basé Debian, est nécessaire pour une partie du séminaire. Il est préférable de connaitre à niveau moyen les concepts du contrôle de sources et/ou avoir utilisé certains de ses logiciels par le passé.Il est aussi nécessaire d’avoir des bases sur le concept d’arborescence des fichiers.

Laboratoire 05 : Utiliser les branches

Pour Windows :

Visualiser les branches disponibles pour le répertoire git

  • Sur la page GitHub le bouton branch :
  • Sur l’application GitHub quand un répertoire est sélectionné le bouton texte en haut de l’historique.
  • Avec Git clic droit sur le répertoire et option branch
  • Sur Git Gui dans branch et Checkout

Créer une nouvelle branche et l’envoyer à GitHub

  • Sur GitHub directement et synchroniser les informations (bouton sync)
  • Sur l’application Git sur le bouton text branch et faire publish
  • Avec Git Gui Branch Create et remplir les champs nécessaire, puis faire un push (façon plus optimale) Visualiser de nouveau vos branches

Changer de branche (Checkout)

  • Par l’interface GitHub en haut de la page
  • En cliquant droit avec Git et choisissant la branche sur le répertoire

Modifier localement votre branche et sauvegarder les changements

  • Supprimer un fichier d’une des branches
  • Faire un push comme appris dans le Lab03

Si un update est nécessaire sur une branche

  • On peut synchroniser avec l’Application client
  • Git Gui cliquer sur remote puis fetch from et finalement origin.

Supprimer une branche en local

  • A partir de Git Gui dans branch delete selectionner la branche et cliquez sur delete
  • Sur l’application client de GitHub cliquer sur les branches et cliquez sur manage, puis cliquer sur la poubelle sur la branche que vous désirez supprimer.

Supprimer une branche sur gitHub

  • Si vous avez utilisé Git Gui il suffit de faire un nouveau push.
  • Si vous avez utilisé GitHub il suffit de faire Sync
  • Si vous n’avez encore rien faire vous pouvez utiliser l’interface web pour supprimer la branche sur GitHub directement. Dans le répertoire branches puis all branches.

Laboratoire 06 : Les organisations (Optionnel)

Durant ce laboratoire vous apprendrez comment créer et gérer les diverses aspects d’une organisation sur GitHub. Une fois ce laboratoire terminé, vous serez en mesure de créer des répertoires, gérer les membres d’une organisation, gérer des groupes et les permissions de ceux-ci.

Créer une organisation

  • Allez sur votre GitHub cliquez sur le + et cliquer new organisation
  • Entrez un nom et un email
  • Sélectionner l’option Open Source
  • Cliquez create puis organization

Inviter des membres

  • Cliquez haut à droite de la page de l’organisation sur le bouton invite someone
  • Entrez le nom d’utilisateur de la personne
  • Sélectionnez le groupe auquel il appartiendra

Créer et gérer une équipe

  • Cliquer à droite dans le menu Team sur Create a new Team
  • Entrez le nom et la description de l’équipe puis choisissez les accès Read, Write ou Admin
  • Cliquez sur Create Team

Créer un répertoire

  • Cliquez sur New Repository sur la page de l’organisation
  • Entrez un nom et une description
  • Cocher une visibilité
  • Ajouter une licence MIT (juste au cas)
  • Cliquez sur create repository

Affecter un répertoire a un groupe

  • Cliquez sur le groupe en question
  • Cliquez sur repositories
  • Dans la barre de recherche add Repositories entrez le nom du répertoire
  • Entrez votre mot de passe dans le pop-up

Rather Drive Stick?

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator written by our own Tom Preston-Werner. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
